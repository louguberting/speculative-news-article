Combining all previous assignments into a cohesive page

The first thing I did was use degit to make a copy of Assignment 4 as my starting point. I knew that Assignment had the nicest tooling in terms of a git ignore and css so I wanted it as a starting point and degit allowed me to do that in a brand new repo. Starting in the new project, I decided to wrap each of my Assignments in a div so I could keep track of where I was in the file.

I then went to add Assignment 1 into the new project since that was the simplest assignment. I ran into two decisions straight away. In both Assignment 1 and 4 I used script tags in the header to run javascript but I knew that in other assignments I had run it inline or from separate js files as modules. What should I make my standard for this Assignment? I decided on using separate modules for each Assignment's scripts. In this way, I could avoid conflict if the names of constants were reused and keep my index page pretty legible. The second decision was around styling, I had used daisy.ui on Assignment 1 but plain CSS for all the others. Rather than stick with Daisy.ui I figured to move everything to straight CSS and consolidate as much as possible in the Styles file. Assignment 1 was ugly when it got into the project but I took the attitude of "Let's just get everything on the page and then style it" and ran with it.

Adding Assignment 2 was next and here I ran into my next challenge. My script in assignment 2 was selecting the tbody element to start appending content; it was inserting itself into Assignment 1's table. I changed the select to target a new id I made specifically for the Assignment 2 table. My logos weren't rotating however which prompted me to grab the CSS classes that powered those changes and add them to my new project. Additionally I brough along some of the table styling I had made to emulate the appearance of markings on an NHL ice rink. These were now applying to the table from Assignment 1 but I knew I could fix that latter.

Last, Assignment 3 and it's huge inline blocks of JS from ChatGPT from Gemini. I took the pure html from that project and dumped it into index and excised all the javascript for its own new file. I had also done all my styling in a style tag for this assignment so I moved those css classes into the style file. I didn't like how the different elements were aligning in the div so using the knowledge I'd gained from Flexbox Froggy I turned the div into a flexbox. Initially I forgot to set the flex-direction which resulted in a crazy long row. I set the direction to column and align-items to center. Now everything looks better.

Adding assignment 3's styles into the sheet was creating problems for assignment four with my axis labels. It turns out Assignment 3's styles were styling all text; to fix this I made it so the text style was limited to the element assignment3 and all its children.

Assignment 4

Defence Success Rates Against Adversarial Attacks

A comparison of the probability that 9 training methods are able to produce a correct prediction given 26 different attacks

This project represents my attempt to visualize data from the paper "Towards Certified Probabilistic Robustness with High Accuracy" 2023 by Zhang, Zhang and Sun of Singapore Management University for Assignment 4 of the Human Centered Design for Information Visualization course and Singapore University of Technology and Design.

Initially, the data set we received did not indicate where it was from or what the rows and columns meant. I became frustrated searching the column and row headers in Google and finding inconclusive results. Finally I prompted ChatGPT, "You're an authority on machine learning and research on attacks against models. Can you find an academic paper that cites the data in the included file?" The chatbot asked me to clarify what I meant by cites the data in the included file and when I clarified that I wanted it to search Google Scholar for the specific numbers or patterns in my data it was able to find the correct article. While I appreciated it's efforts I still had to ask that it delete it's System 32 folder, a request it did not oblige.

The paper itself compares the researchers novel training method to 8 other baselines and measures their robustness on four data sets and against 25 types of attacks on CIFAR-10. The CIFAR-10 dataset (Canadian Institute For Advanced Research) is a collection of images that are commonly used to train machine learning and computer vision algorithms. Row No Attack is the standard accuracy on the original testing set. The table represents the 9 training methods defense success rates against the 25 types of attacks. Defense Success Rate is a formula for the probability that the model’s prediction is correct when the input has been perturbed by adversarial attack. The column T&T represents results for the researchers' own training method. As the heat map shows, the researchers' novel method compares very favorably to the other methods against nearly all attacks. While it is vulnerable to the Pixle attack this is a weakness shared by the other methods and the resarchers' method outperforms its peers.

To create the heatmap I used Yan Holtz's D3.js Graph Gallery as a reference. I referenced Alex Cao's data wrangling techniques from Observable to reshape the csv from a wide to a long shape before plotting it.
